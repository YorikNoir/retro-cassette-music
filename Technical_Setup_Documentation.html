Retro Cassette Music - Technical Setup

What Was Built

  • Django web application with retro cassette UI
  • RESTful API with JWT authentication
  • AI music generation (ACE-Step v1.5)
  • LLM lyrics generation (local or OpenAI)
  • User library with voting/publishing
  • Celery async tasks for generation

Tech Stack

  • Django 5.0.1 + DRF 3.14.0
  • Vanilla JavaScript (no frameworks)
  • PyTorch 2.10 + Transformers 5.1
  • Celery 5.3.4 + Redis 5.0.1
  • JWT authentication + encrypted API keys
  • SQLite (dev) / PostgreSQL (prod)

Key Files

Startup Scripts

  • start.bat → Windows startup
  • start.sh → Linux/Mac startup
  • stop.bat → Windows shutdown
  • stop.sh → Linux/Mac shutdown

Configuration

  • .env → Environment settings
  • config/settings.py → Django config
  • requirements.txt → Python deps

Commands

Start All Services (Windows):
start.bat
Start All Services (Linux/Mac):
chmod +x start.sh && ./start.sh
Stop All Services (Windows):
stop.bat
Stop All Services (Linux/Mac):
chmod +x stop.sh && ./stop.sh

Live URLs

  • Local Server: http://localhost:8000
  • GitHub Repo: https://github.com/YorikNoir/retro-cassette-music
  • API Docs: http://localhost:8000/api/ (when running)

Highlights

Project Structure

Backend Apps

  • accounts/ User auth, profiles
  • songs/ Song CRUD, voting
  • generation/ AI generation tasks
  • library/ User statistics

Frontend

  • templates/ HTML (SPA)
  • static/css/ Retro styling
  • static/js/ Vanilla JS logic

Major Features

  • Auto-startup: Both Django and Celery start with one command
  • Cross-platform: Works on Windows, Linux, and macOS
  • Encrypted API keys: Secure storage for OpenAI credentials
  • Async generation: Non-blocking music creation via Celery
  • Retro UI: Cassette-themed player with animations

Key Solutions

Challenge: Manual Celery startup required
Solution: Auto-launch in separate window/background process

Challenge: Fernet encryption key generation
Solution: Python cryptography.fernet.Fernet.generate_key()
